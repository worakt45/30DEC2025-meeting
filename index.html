  • ไทม์ไลน์ฉบับปรับปรุง: เริ่มทดสอบระบบนำร่อง (Pilot testing) กับผู้ให้เช่าภายในวันที่ 7 มกราคม (เลื่อนเข้ามาจากเดือนมีนาคม)

  • ช่องทางการชำระเงิน: จะตัดสินใจเลือกระหว่าง Omise หรือ 2C2P หลังจากเสร็จสิ้นเฟสนำร่อง (หลัง 7 ม.ค.)

MVP Demonstration - Front-End

  • Roles: Supports 5 roles: Admin, Landlord, Manager (Agents), and Tenants.
  • Admin Features: System analytics, transactions, property management, and finance overview.
  • Services Tab: Integration with home services (Home Pro), repairs, cleaning, and worker availability status.
  • Settings: Theme options (light/dark), language, and time zone.
  • Landlord View: Focuses on property management, events, finances, and service requests.

การสาธิต MVP - ระบบหน้าบ้าน (Front-End)

  • บทบาทผู้ใช้: รองรับ 5 ประเภท: ผู้ดูแลระบบ (Admin), เจ้าของที่พัก (Landlord), ผู้จัดการ/นายหน้า, และผู้เช่า
  • ฟีเจอร์ Admin: ตัวชี้วัดระบบ, ธุรกรรมล่าสุด, กราฟวิเคราะห์, การจัดการทรัพย์สิน และภาพรวมการเงิน
  • แท็บบริการ: เชื่อมต่อกับแพลตฟอร์มบริการเกี่ยวกับบ้าน (เช่น Home Pro), การซ่อมแซม, ทำความสะอาด และสถานะงานของช่าง
  • การตั้งค่า: ปรับแต่งธีม (โหมดมืด/สว่าง), ภาษา และโซนเวลา
  • มุมมองเจ้าของที่พัก: เน้นการจัดการทรัพย์สิน, ติดตามเหตุการณ์, การเงิน และคำร้องขอรับบริการ

MVP Demonstration - Back-End

  • Foundation: Designed for stability to avoid future core structural changes.
  • Security (Fuzzy Location): Shows approximate coordinates (like Facebook/Apple) to protect landlord privacy.
  • Integrations: APIs for registration, login, and LINE messenger.
  • LINE Notification: Using LINE Official Account for push notifications as web apps lack native push capabilities.

การสาธิต MVP - ระบบหลังบ้าน (Back-End)

  • โครงสร้างพื้นฐาน: ออกแบบให้แกนหลักมีความเสถียร เพื่อลดการแก้ไขโครงสร้างใหญ่ในอนาคต
  • ความปลอดภัย (Fuzzy Location): แสดงพิกัดโดยประมาณ (คล้าย Facebook/Apple) เพื่อปกป้องความเป็นส่วนตัวของที่ตั้งจริง
  • การเชื่อมต่อ: มี API สำหรับการลงทะเบียน, เข้าสู่ระบบ และเชื่อมต่อกับ LINE
  • การแจ้งเตือนผ่าน LINE: ใช้ LINE Official Account สำหรับการแจ้งเตือน เนื่องจากเว็บแอปพลิเคชันไม่สามารถส่ง Push Notification ได้เหมือนแอปมือถือ

Key Innovation: Airbnb Import

Rationale: Removes friction for landlords with multiple properties (20+ units).

  • One-Click Migration: Uses crawling/RPA to import data from Airbnb listings.
  • Automated: Extracts details and uploads photos to servers automatically.
  • Impact: Expected to significantly increase conversion rates.

นวัตกรรมหลัก: การดึงข้อมูลจาก Airbnb

เหตุผล: ลดความยุ่งยากสำหรับเจ้าของที่มีห้องพักจำนวนมาก (20+ ยูนิต) ที่ไม่อยากกรอกข้อมูลใหม่

  • ย้ายข้อมูลในคลิกเดียว: ใช้เทคโนโลยี Crawling/RPA เพื่อดึงข้อมูลจากรายการใน Airbnb โดยอัตโนมัติ
  • ระบบอัตโนมัติ: ดึงรายละเอียดและอัปโหลดรูปภาพขึ้นเซิร์ฟเวอร์ทันที
  • ผลลัพธ์: คาดว่าจะช่วยเพิ่มอัตราการเปลี่ยนผู้เยี่ยมชมให้เป็นลูกค้า (Conversion) ได้อย่างมีนัยสำคัญ

Strategy & Target Market

  • Target: Entire home rentals (Airbnb revenue in TH is 800M THB/year).
  • Goal (18-Month): 50M THB Revenue (6.25% market share).
  • Service Platform: Connect via API to existing providers (Q-Chang, Home Pro) for commission (2-5%). Avoid in-house teams initially.
  • Advantage: Scraping Airbnb contacts for direct marketing.

กลยุทธ์และกลุ่มเป้าหมาย

  • เป้าหมายหลัก: บ้านพักทั้งหลัง (รายได้ Airbnb ในไทยรวม 800 ล้านบาท/ปี)
  • เป้าหมาย 18 เดือน: รายได้ 50 ล้านบาท (ส่วนแบ่งตลาด 6.25%)
  • แพลตฟอร์มบริการ: เชื่อมต่อ API กับผู้ให้บริการเดิม (Q-Chang, Home Pro) เพื่อรับค่าคอมมิชชั่น (2-5%) หลีกเลี่ยงการจ้างทีมช่างเองในช่วงแรก
  • ข้อได้เปรียบ: ความสามารถในการดึงข้อมูลติดต่อจาก Airbnb เพื่อทำการตลาดโดยตรง
